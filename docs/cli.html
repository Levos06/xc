01 · CLI

xc-cli

Детерминированная, локальная утилита командной строки — ядро экосистемы. Извлечение кода тривиально и работает за линейное время, а результат побитово стабилен при любой правке прозы.

Посмотреть формат → Исходники ↗
Команды

Четыре операции.

xc-cli init

Оборачивает обычный исходник (.py, .ts, …) в структуру .xc: монолитный блок кода и стартовое объяснение.

xc-cli extract

Поточно отдаёт чистый исполняемый код за O(n). Флаг --hash печатает SHA-256 слоя кода.

xc-cli run

Извлекает код и запускает его соответствующим интерпретатором (Python, Node, Ruby, Bash…).

xc-cli validate

Проверяет структуру и YAML, устойчив к обрыву стрима от LLM — сообщает строку, до которой файл восстановим.

Как пользоваться

Установка и запуск.

# ядро + CLI
cd core
python3 -m venv .venv
.venv/bin/pip install -e .

# обернуть существующий файл
xc-cli init mymodule.py        # → mymodule.xc

# работать с .xc
xc-cli validate examples/auth.xc
xc-cli extract  examples/auth.xc --hash
xc-cli run      examples/auth.xc
No diff noise

Прозу не видно на код-ревью.

Скрипт git-integration/setup-xc-diff.sh ставит textconv-драйвер (xc-cli extract) и .gitattributes: git diff по .xc показывает только слой кода. Правка только описания даёт пустой diff.

git-integration/setup-xc-diff.sh /path/to/repo
