Код и смысл —
в одном файле

.xc хранит исполняемый код вместе с его смысловым слоем — архитектурным контекстом, инвариантами и требованиями — так, чтобы это оставалось понятным людям, инструментам и языковым моделям.

Зачем

Документация внутри кода — это старо. Сделать её надёжной для автоматики — нет.

Три инженерных принципа лежат в основе формата и проверяются тестами.

🧱

Изоляция слоёв (LangSec)

Один построчный, тотальный распознаватель за O(n). Код внутри блока никогда не парсится как разметка и наоборот — никаких «двух парсеров».

🧭

Planning Conditioning

Объяснение и инварианты физически идут раньше кода. Модель сначала фиксирует план и граничные случаи, затем пишет реализацию.

🔇

Без diff-шума

Извлечение кода зависит только от блока MONOLITH: правки прозы не меняют код и его хэш. В git правка текста даёт пустой diff.

Формат v2

Монолитный код + диапазоны строк

Код — один блок в конце файла. Блоки объяснений ссылаются на 1-индексные строки кода маркером lines: (диапазоны могут пересекаться).

---
xc_spec: "2.0"
language: "python"
module: "core.auth"
---

# [EXPLANATION: overview]
lines: 1-6
## Архитектурный контекст
Проверяет сессионный токен перед обработкой запроса.

# [EXPLANATION: fail_closed]
lines: 4-5
## Инвариант безопасности
Токен без поля `exp` отвергается (fail-closed).

# [CODE: MONOLITH]
```python
import time

def verify_session(token: dict) -> bool:
    if "exp" not in token:
        return False
    return token["exp"] > time.time()
```
Вьюер

Два режима отображения

Так это выглядит в расширении для VS Code / Cursor. Наводите курсор на строки кода, переключайте режимы, в «Сетке» — кликайте по ячейке, чтобы раскрыть её целиком.

auth.xc

В «Контексте» карточка появляется при наведении на строку. В «Сетке» строки справа выровнены со строками кода 1:1; блок занимает строки до начала следующего, клик раскрывает его целиком.

Экосистема

Три инструмента, один парсер

CLI, инструменты для агентов и расширение редактора используют единый распознаватель — Python и TypeScript согласны побайтово о том, где находится код.

core/ · Python

xc-cli

  • init — обернуть исходник в .xc
  • extract — чистый код (O(n))
  • run — извлечь и выполнить
  • validate — структура + YAML, устойчивость к обрыву
mcp/ · MCP

Инструменты агентов

  • extract_code_layer
  • update_explanation_block — правка прозы, хэш кода неизменен
  • generate_explained_artifact — план-первый
  • explanation_gate — Teach-Back барьер (SOLO)
vscode-extension/ · TS

Расширение редактора

  • Сплит: код + проза, два режима
  • Подсветка, номера строк, формулы (MathML)
  • Правка/создание/удаление блоков, undo
  • Изолированный git-diff по слоям
Старт

Попробовать локально

Требуется Python 3.9+ и Node 18+.

CLI и MCP

# ядро + CLI
cd core
python3 -m venv .venv
.venv/bin/pip install -e .

# попробовать
xc-cli validate examples/auth.xc
xc-cli run examples/auth.xc

Расширение

# установить собранный пакет
code --install-extension \
  vscode-extension/versions/explained-code-2.2.0.vsix

# или для разработки
cd vscode-extension
npm install && npm run compile
