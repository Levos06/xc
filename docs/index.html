Код и смысл —
в одном файле.

.xc хранит исполняемый код вместе с его смысловым слоем — архитектурным контекстом, инвариантами и требованиями — так, чтобы это оставалось понятным людям, инструментам и языковым моделям.

Живое демо → В чём идея
Идея

Документация и код перестают расходиться.

В обычном проекте смысл живёт отдельно от кода — в вики, в головах, в устаревших комментариях. .xc делает смысловой слой частью файла: один монолитный блок кода, а над ним — объяснения, привязанные к конкретным строкам.

Код остаётся чистым и компилируемым. Объяснения можно перекрывать, вкладывать и комментировать построчно — не трогая ни байта кода.

Философия

Три принципа, которые делают формат надёжным.

Это не лозунги — каждый из них заложен в реализацию и проверяется тестами.

01

Изоляция слоёв

Один построчный, тотальный распознаватель за O(n). Код внутри блока никогда не парсится как разметка и наоборот — никакой патологии «двух парсеров» (LangSec).

02

Planning Conditioning

Объяснения и инварианты физически расположены раньше кода. Модель сначала фиксирует план и граничные случаи, и лишь затем пишет реализацию.

03

Без diff-шума

Извлечение кода зависит только от блока кода: правка прозы не меняет ни строки кода, ни его хэш. В git правка текста даёт пустой diff.

Экосистема

CLI, инструменты для агентов и расширение редактора используют единое ядро — Python и TypeScript согласны побайтово о том, где находится код.
