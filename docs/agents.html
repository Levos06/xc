02 · MCP

Инструменты
для агентов.

Набор атомарных операций для автономных ИИ-агентов через Model Context Protocol. Каждый инструмент также импортируется как обычная Python-функция — для LangChain или любого хоста.

Четыре инструмента

Безопасная работа агента с .xc.

extract_code_layer

Возвращает только исполняемый код — готовый для компилятора или интерпретатора.

update_explanation_block

Изолированно переписывает одно объяснение и возвращает доказательство, что хэш кода не изменился.

generate_explained_artifact

Собирает валидный .xc из монолитного кода и объяснений с диапазонами строк — отказываясь выдавать код без объяснения (Planning Conditioning).

explanation_gate

Когнитивный барьер Teach-Back: модель-судья (T=0.1) оценивает объяснение человека по таксономии SOLO и разрешает мердж только выше порога.

Teach-Back

Барьер против «vibe coding».

Если код сгенерирован ИИ, explanation_gate просит человека объяснить логику и граничные условия своими словами. Локальная модель-судья оценивает ответ по SOLO-таксономии и пропускает мердж только тогда, когда понимание реально продемонстрировано. Без ключа API работает прозрачная офлайн-эвристика.

Подключение

Запустить MCP-сервер.

Регистрируется в Cursor или Claude Desktop как обычный MCP-сервер.

cd mcp
../core/.venv/bin/pip install "mcp>=1.2.0"   # опц.: anthropic для судьи
../core/.venv/bin/python -m xc_mcp.server     # stdio-транспорт

Пример прямого вызова из Python/LangChain — в mcp/examples/langchain_client.py.
